# Guia de Suporte Alpha: Talkvex

Este guia orienta o time de MKT e Customer Success sobre como lidar com as interações durante a fase Alpha do Talkvex.

## 1. Princípios de Atendimento
*   **Transparência Alpha:** Sempre admita que estamos em Alpha. Usuários pioneiros adoram ajudar a construir o produto, não esconda bugs.
*   **Velocidade de Resposta:** No Alpha, o suporte deve ser quase instantâneo para manter a confiança.
*   **Persona "Coach":** Mantenha o tom de um mentor que está aprendendo junto com o usuário.

## 2. Respostas para Cenários Comuns

### Cenário A: Bug Reportado (ex: Streak não atualizou)
*   **Resposta:** "Opa! Obrigado pelo report. Como estamos em Alpha, esses pequenos desajustes podem ocorrer. Já notifiquei o time de engenharia ([Link para Ticket]) e estamos cuidando disso. Sua dedicação não será perdida!"
*   **Ação:** Criar ticket para DEV se ainda não existir.

### Cenário B: Pedido de Feature (ex: Integração com Calendário)
*   **Resposta:** "Excelente ideia! Essa é uma das funções mais pedidas pela nossa comunidade Alpha. Ela já está no nosso roadmap de curto prazo. Fique de olho nas atualizações!"
*   **Ação:** Registrar no log de métricas como "Demanda Repetitiva".

### Cenário C: Elogio/Sucesso (ex: Concluí meu primeiro Streak)
*   **Resposta:** "Parabéns pela consistência! 🚀 O primeiro Streak é o mais difícil. Continue assim, o ritual é o que traz o resultado!"
*   **Ação:** Pedir permissão para usar como depoimento (Prova Social).

## 3. Fluxo de Escalação
1.  **Dúvida de Uso:** MKT responde usando a `HELP_DOCUMENTATION.md`.
2.  **Bug Técnico:** MKT abre ticket para DEV e informa o usuário.
3.  **Sugestão Estratégica:** MKT registra em `learnings_internal.md` para revisão do CEO (KERNEL).
