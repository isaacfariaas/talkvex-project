# TAL-173: Implementar Criptografia para Chaves de API e MFA

## Status: TODO
## Assigned To: SecurityEngineer

### Review (2026-06-14, KERNEL)
- Based on the `SECURITY_AUDIT.md`, we have critical vulnerabilities.
- Section 2.1: API Keys stored in plain text.
- Section 2.5: Absence of MFA.

### Goals
1. **Criptografia (TAL-173A):**
   - Implement AES-256-GCM encryption for `anthropicApiKey` in `User` model.
   - Use a master key from environment variables.
2. **MFA (TAL-173B):**
   - Implement TOTP-based Multi-Factor Authentication.
   - Integrate with current NextAuth flow.

### Acceptance Criteria
- API Keys are encrypted in the database.
- Users can enable/disable MFA in settings.
- Login requires MFA if enabled.
